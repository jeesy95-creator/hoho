## 문제 요약
아날로그 시계에서

- **초침이 분침 또는 시침과 겹칠 때마다 알람이 울린다.**
- 시작 시각 `(h1, m1, s1)`부터 종료 시각 `(h2, m2, s2)`까지
  **(시작 시각 포함)** 알람이 울린 횟수를 구한다.
- 단, **0시 / 12시 정각처럼 초침·분침·시침이 동시에 겹치는 경우에도 알람은 1번만** 울린다.

---

## 내가 처음에 틀린 이유

1. `solution()` 함수가 없어 채점 실패
2. 누적 계산 함수 안에서 구간 계산을 수행함 (함수 역할 혼합)
3. 외부 변수(`h1, m1, s1`)를 함수 내부에서 참조
4. 시작 시각 포함 여부 처리 누락
5. 0시 / 12시 정각의 **3침 동시 겹침 중복 제거 미흡**

→ 일부 테스트에서 **+2 오차 발생**

---

## 풀이 1️⃣ (정답) — 수학적 누적 카운팅 O(1)

### 핵심 아이디어
- 시간 `t`초까지의 **누적 알람 횟수 F(t)** 를 수식으로 계산
- 구간 알람 횟수:


### 수식
- 초침 ↔ 분침 겹침  
`floor(59 * t / 3600) + 1`
- 초침 ↔ 시침 겹침  
`floor(719 * t / 43200) + 1`
- 3침 동시 겹침(0시, 12시) 중복 제거  
`(t // 43200) + 1`

### 시간 복잡도
- **O(1)**

---

## 풀이 2️⃣ — 1초 단위 완전 시뮬레이션 O(86400)

### 아이디어
- 시작 시각부터 종료 시각까지 **1초씩 증가**
- 각 초마다:
- 초침-분침 겹침 여부
- 초침-시침 겹침 여부
- 둘 중 하나라도 만족하면 알람 +1  
(둘 다 만족해도 1번만)

### 장점
- 가장 직관적
- 디버깅 / 검증용으로 매우 좋음

---

## 풀이 3️⃣ — 이벤트 시각 병합(Merge) 시뮬레이션

### 아이디어
- 초침-분침 겹침 시각: `k * 3600 / 59`
- 초침-시침 겹침 시각: `k * 43200 / 719`
- 두 수열을 **merge** 하면서 구간 내 이벤트만 카운트
- 같은 시각(3침 겹침)은 1번만 카운트

### 특징
- 수학 풀이보다 직관적
- 이벤트 개수(≈ 2852개)만 순회 → 충분히 빠름
